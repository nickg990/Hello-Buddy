# 6. PDF Programme Requirements

## Purpose

The generated PDF is the primary Release 1 output. It replaces the current manual process of creating exercise programme attachments and sending them to pet owners via WhatsApp or similar channels.

The PDF must be clear enough for an owner to follow without needing access to the admin system.

## Output rule

The PDF must be generated from structured database records, not manually typed from scratch each time.

## PDF content

### Header

Include:

- Hello Buddy logo;
- practice or service name if available;
- programme title;
- pet name;
- owner name;
- practitioner name;
- programme start date;
- programme end date;
- generated/published date;
- version number if useful.

### Introductory text

Include a short friendly instruction block, for example:

- follow the programme as advised by your physiotherapist;
- use the linked videos to check technique;
- stop and contact the practice if the dog appears uncomfortable or symptoms worsen;
- do not increase repetitions or intensity without clinical advice.

The exact wording can be refined later.

### Programme sections

The PDF should support either:

- one daily session; or
- AM and PM sessions.

Each session should show:

- session heading;
- optional objective/notes;
- ordered list of exercises.

### Exercise content

Each exercise should show:

- exercise title;
- image or thumbnail where available;
- short summary;
- reps;
- sets;
- hold seconds if relevant;
- frequency or timing if relevant;
- practitioner notes if entered;
- ordered instructions;
- clickable video link.

The video link should be user-friendly. Prefer link text such as "Watch video" or the exercise name rather than a raw URL, while ensuring the URL remains visible or accessible if printed.

### Footer

Include:

- practice contact details if available;
- page number;
- disclaimer or safety note if approved;
- generated date/version metadata in small text.

## PDF layout

Recommended layout:

1. Cover/header block with pet and programme details.
2. Safety/instruction block.
3. Session section.
4. Exercise cards or clean rows.
5. Footer.

Use the Hello Buddy palette but keep print readability high.

## Versioning rules

Publishing a programme must:

- validate required data;
- create an immutable `ProgrammeVersion`;
- generate the final PDF;
- store the PDF file reference/path;
- mark the version as current where appropriate;
- not overwrite an existing published version.

If a practitioner changes a programme after publishing, the system should create a new draft and publish it as a new version.

## Draft preview

The practitioner must be able to preview the PDF before publishing.

The preview should:

- render inside the admin preview page rather than as a separate downloadable draft file;
- show the same structure as the final PDF;
- highlight missing video links or missing prescription values;
- allow the practitioner to return to editing;
- prevent publishing if critical required fields are missing.

The draft preview must not offer print or download actions before publish, to avoid confusion between a draft view and the final issued owner document.

## Validation rules

Minimum validation before publishing:

- owner exists;
- pet exists;
- treatment case exists;
- programme has title or identifiable date range;
- programme has start and end date;
- programme has at least one session;
- each session has at least one exercise;
- each exercise has title and prescription values;
- missing video links are shown as warnings or errors depending on final policy.

## File storage

The PDF file may be stored in local storage during early development and moved to Azure Blob Storage or equivalent in production.

The database should store metadata such as:

- programme version reference;
- file name;
- storage path or blob key;
- generated date/time;
- generated by;
- file type;
- file size if available.

## Future compatibility

The same structured programme data should later support:

- mobile JSON programme snapshots;
- owner app display;
- XML export for vets;
- analytics on exercise/programme usage.
