# TD-001 — Admin solution deviations from `Standards/coding-standards.md`

**Raised:** 2026-05-28 (end of Day 2, MSc Assessment 3 build week)
**Scope:** `Canine Physio Admin/` (ASP.NET Core admin web app deployed for the COM712 infrastructure assessment)
**Context:** The MSc submission is an **infrastructure** assessment — the marker is grading the Azure architecture, deployment, identity, secrets, observability and cost story, not the codebase. The admin app exists to give the cloud a real workload to host. After submission this codebase becomes the basis for the production Canine Physio Admin solution, at which point the deviations below must be paid down before any second feature lands.

This file replaces the per-deviation ADRs that `coding-standards.md` would otherwise require. Each row carries a remediation owner of **"post-assessment, Increment 2 kickoff"** unless stated otherwise.

---

## Decision

For the duration of MSc Assessment 3 (build complete by Sun 1 Jun 2026, submission Mon 2 Jun) the admin app is permitted to ship in its current shape. The "prototype escape hatch" in §2 of the coding standards (_"For pure prototyping (Increment 0/1 design-validation), put everything in `HelloBuddy.Web` and lift it out as boundaries crystallise"_) is invoked across the board. After submission, **no new feature work** starts until the items flagged "Increment 2 blocker" below are cleared.

---

## Critical fixes still required before Friday lockdown

These are not deferred — they must be done as part of the Day 3 / Day 4 Azure work because they block production behaviour, not because of standards purity.

| #   | Item                                                                                                                                       | Why it can't wait                                                                                                                                                                                       |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| C1  | Builder.cshtml renders the literal text `rowIndex++;` because a `@{ }` block was lost during formatting; closing tags are also mis-nested. | Real UI bug in a demoed page.                                                                                                                                                                           |
| C2  | No `IFileStore` abstraction — `ProgrammesController.Publish` calls `File.WriteAllBytesAsync` directly.                                     | Day 4 swaps the PDF target to Azure Blob. Without `IFileStore` that's a controller edit instead of a DI swap, and the local/cloud parity story in the demo collapses.                                   |
| C3  | `Program.cs` does not call `builder.Configuration.AddAzureKeyVault(...)`.                                                                  | Day 3 Container App reads its MySQL connection string and App Insights connection string from Key Vault via managed identity. Required for the assessment to demonstrate "no secrets in config or env". |
| C4  | Dev MySQL password (`Pwd=devroot`) is committed in `appsettings.Development.json`.                                                         | Establishes the wrong muscle memory for the team and is a trivial fix (`dotnet user-secrets`).                                                                                                          |
| C5  | `Views/Shared/_Layout.cshtml` uses `lang="en"` and has no skip-to-content link; Builder form inputs have no `<label for>`.                 | WCAG 2.2 AA is a release gate (§19) and these four lines of HTML take 10 minutes. The assessment Demo Video Script references accessibility.                                                            |

All other items in this file are deferred.
